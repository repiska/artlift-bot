# Руководство по внесению вклада

Спасибо за интерес к проекту Art Lift Community Bot!

## Процесс разработки

1. Создайте форк репозитория
2. Создайте ветку для вашей функции (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Добавьте тесты для новых функций
5. Убедитесь, что все тесты проходят (`pytest`)
6. Закоммитьте изменения (`git commit -m 'Add some amazing feature'`)
7. Запушьте в ветку (`git push origin feature/amazing-feature`)
8. Откройте Pull Request

## Стандарты кода

### Python

- Используйте Python 3.11+
- Следуйте PEP 8
- Используйте type hints где возможно
- Документируйте функции и классы

### Коммиты

Используйте понятные сообщения коммитов:
- `feat:` новая функция
- `fix:` исправление бага
- `docs:` изменения в документации
- `test:` добавление или изменение тестов
- `refactor:` рефакторинг кода
- `chore:` обновление зависимостей, конфигурации

Пример: `feat: add reminder scheduling functionality`

### Тестирование

- Все новые функции должны иметь тесты
- Цель покрытия кода: минимум 80%
- Тесты должны быть независимыми и быстрыми

### Структура проекта

```
bot/
├── handlers/      # Обработчики событий
├── services/      # Бизнес-логика
├── keyboards/     # UI компоненты
├── middlewares/   # Middleware
├── database/      # Работа с БД
└── utils/         # Утилиты

tests/             # Тесты
config/            # Конфигурация
```

## Перед отправкой PR

- [ ] Код соответствует стандартам PEP 8
- [ ] Все тесты проходят
- [ ] Добавлены тесты для новых функций
- [ ] Обновлена документация (если нужно)
- [ ] Код прошел линтер (`flake8`, `black`, `isort`)
